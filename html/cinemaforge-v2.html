놓친 핵심 모델 3개
  • Qwen/Qwen-Image-Edit-2511 + fal/Qwen-Image-Edit-2511-Multiple-Angles-LoRA — 이게 빠지면 카메라 포즈→뷰 생성 파이프라인의 핵심이 없음. 96 포즈, Gaussian Splatting 데이터 학습, Three.js 뷰포트 이미 존재 (ComfyUI-qwenmultiangle). Apache-2.0.
  • stabilityai/stable-virtual-camera (SEVA) — 2025년 3월 릴리즈, 단일 이미지→카메라 궤적 영상 생성 SOTA. 1.3B 모델로 가볍고 14개 카메라 패스 내장. 단 비상업 라이선스 주의.
  • Wan-AI/Wan2.1-VACE-14B — Wan 2.1 단순 I2V가 아닌 VACE(비디오 애니메이션 컨트롤 엔진). 카메라 트래젝토리, 참조 영상 기반 편집. Apache-2.0.
잘못 제안된 항목들
  • TencentARC/InstantMesh → 기하학 품질 불안정, 실제 ArchViz에 쓰기 어려움. Qwen Multi-Angle로 대체.
  • naver-labs-europe/MASt3R → 멀티뷰 3D 재구성용. 우리 MVP에는 과도함. 나중 단계로 이동.
  • Wan-AI/Wan2.1-I2V-14B → 이미 Wan2.1-VACE-14B와 Wan2.2-I2V-A14B로 업그레이드됨. 구버전.
  • CameraCtrl II → ICCV 2025 논문이지만 weights 공개 미확인. 포함 불가.
참조 프로젝트 발견

Triplex (pmndrs/triplex) — React Three Fiber 기반 오픈소스 비주얼 워크스페이스. 2025년 10월 완전 오픈소스 전환. CinemaForge 3D 편집 UI의 직접적 레퍼런스. ComfyUI-qwenmultiangle도 이미 Three.js 뷰포트 + 카메라 각도 제어를 구현함 — 우리가 만들려는 것의 UI 패턴 직접 참조 가능.

현실적 하드웨어 요구사항

구성GPU가능한 모델제한사항
MVP 최소 RTX 3090 / 4090
24GB VRAM		 Qwen + LoRA, SEVA, SAM2, Depth-Anything, Wan2.1-VACE-14B FLUX.1-dev 동시 실행 불가, Wan2.2 불가. 모델 스왑 필요.
권장 풀스택 A100 / H100 40-80GB
40GB+ VRAM		 전체 모델 동시 로딩 가능 —
저사양 대안 RTX 3080 16GB Qwen + LoRA (16GB), SAM2, Depth-Anything Wan 계열 불가 → SEVA만 (8GB). 영상 생성 API 대체 필요.
개발/테스트 CPU만 3D UI 개발, API 목업 AI 기능 전부 불가. 목업 응답으로 UI 개발 가능.
24GB 단일 GPU MVP 로딩 전략

동시 로딩 대신 작업별 모델 스왑: Qwen(16GB) → SAM2(6GB) → Wan VACE(24GB) 순서로 load/unload. torch.cuda.empty_cache() + gc.collect() 호출. bitsandbytes 4bit 양자화로 Wan VACE를 14→8GB로 줄일 수 있음.

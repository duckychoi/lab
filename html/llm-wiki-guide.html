Raw Sources

원본 소스 문서들. 기사, 논문, 메모, 데이터 파일. 불변 — Claude는 읽기만 하고 절대 수정하지 않는다. 진실의 소스.

raw/ 디렉토리 구조
방금 생성된 폴더 — 이렇게 채워나가면 된다
📁 Obsidian_WS / raw/
raw/ 방금 생성 ├── assets/ 이미지 전용 │ └── .gitkeep ├── articles/ ← 기사, 블로그 포스트 (.md) ├── papers/ ← 논문, 기술 리포트 (.md, .pdf) ├── notes/ ← 직접 쓴 메모, 대화 요약 (.md) ├── data/ ← 데이터 파일, 스프레드시트 └── README.md 생성됨
⚠️ 기존 vault 파일들도 사실상 raw sources — 이렇게 분류된다
Obsidian_WS/ ├── AI News/ ← AI 뉴스 클리핑들 raw source ├── Cinema Studio/ ← 시네마 스튜디오 자료 raw source ├── Vibe Coding/ ← 바이브 코딩 자료 raw source ├── Research.md ← Qwen 연구 노트 raw source ├── design.md ← HTML 디자인 가이드 raw source │ ├── raw/ ← 앞으로 새 소스는 여기 신규 ├── wiki/ ← Claude가 관리하는 wiki └── CLAUDE.md ← 스키마
raw/ vs wiki/ — 핵심 차이
왜 두 폴더가 분리되어 있는가
raw/ — 원본 소스
✗ Claude가 수정하지 않음
✓ 사용자가 직접 파일 추가
✓ 원문 그대로 보관
✓ 진실의 소스
✓ 이미지, PDF, 데이터 모두 OK
✓ 삭제해도 wiki는 살아있음
예: 기사 마크다운, 논문 PDF, 메모
wiki/ — 합성 지식
✓ Claude가 소유하고 관리
✗ 사용자가 직접 수정 안 함
✓ 요약, 크로스 레퍼런스, 합성
✓ 마크다운 + wikilinks
✓ 소스 추가할수록 더 풍부해짐
✓ 쿼리 답변의 기반
예: concept 페이지, entity 페이지, 도메인 종합
raw/ 사용 워크플로우
새 소스를 추가하고 wiki에 반영하는 전체 과정
1
소스 준비 — raw/에 파일 넣기
방법 A — Obsidian Web Clipper로 기사 클리핑
→ raw/articles/[제목].md 자동 저장

방법 B — 파일 직접 드래그앤드롭
→ raw/papers/, raw/notes/ 등에 저장

방법 C — 기존 폴더 파일 (AI News/, Cinema Studio/, Vibe Coding/)
→ 그대로 사용 가능, raw source로 취급됨
2
Claude에게 인제스트 요청
Claude Code에서 /ingest 스킬 실행 후:

"raw/articles/[파일명].md 인제스트해줘"
"AI News/[파일명].md wiki에 추가해줘"
"[URL] 인제스트해줘" (URL 직접도 가능)

→ 배치 인제스트도 가능: "raw/articles/ 안에 있는 거 다 인제스트해줘"
3
Claude가 wiki 업데이트 (자동)
Claude가 알아서 처리하는 것들:
→ wiki/sources/[slug].md 요약 페이지 생성
→ wiki/index.md 업데이트
→ 관련 wiki/concepts/, wiki/entities/ 페이지 생성/업데이트
→ wiki/domains/ 도메인 페이지 업데이트
→ wiki/log.md에 기록

소스 1개 → 보통 5~15개 wiki 파일에 영향
4
Obsidian에서 결과 확인
Obsidian Graph View로 wiki 토폴로지 확인
→ 새 페이지들이 어떻게 연결됐는지 시각화

wiki/index.md 읽어서 전체 목록 확인
wiki/log.md에서 방금 한 인제스트 기록 확인

궁금한 거 있으면: /query "질문"
raw/ 사용 규칙
지키면 wiki가 깔끔하게 유지된다
✅
파일은 그대로 넣기
원문을 수정하거나 요약하지 않고 그대로 저장. 요약은 Claude가 wiki에서 담당.
🚫
raw/를 직접 편집하지 않기
Claude도, 사용자도 raw/ 파일을 나중에 수정하면 안 됨. 추가는 OK, 수정은 NO.
📥
이미지는 raw/assets/에
Obsidian 설정에서 attachment 폴더를 raw/assets/로 설정하면 자동 저장됨.
🚫
wiki/에 원본 넣지 않기
wiki/는 합성 레이어. 원본 파일을 여기에 넣으면 레이어 구분이 무너짐.
💡
인제스트 전에 미리 읽어도 OK
raw/ 파일을 Obsidian에서 직접 읽어도 됨. 인제스트는 Claude가 wiki에 반영할 때만 필요.
🔗
기존 폴더도 raw source
AI News/, Cinema Studio/, Vibe Coding/ 폴더 파일들도 raw source. 새 파일은 raw/에 추가하는 게 더 정리됨.
Obsidian 설정 — raw/assets/ 자동 저장
Web Clipper로 가져온 이미지가 raw/assets/에 자동 저장되도록
1
Attachment 폴더 설정
Settings → Files and links → Default location for new attachments → In the folder specified below
Attachment folder path: raw/assets
2
이미지 다운로드 단축키 설정 (선택)
Settings → Hotkeys → "Download attachments for current file" 검색
권장 단축키: Ctrl+Shift+D — Web Clipper 후 한 번에 이미지 다운로드
3
Obsidian Web Clipper 설치
Chrome / Firefox 확장 프로그램 → "Obsidian Web Clipper" 검색
기사 클리핑 시 저장 위치를 raw/articles/ 또는 AI News/로 지정
🗂️

한 줄 요약

raw/는 원본을 보관하는 곳. 사용자가 파일을 넣고, Claude에게 /ingest를 요청하면 Claude가 wiki/를 자동으로 업데이트한다. raw/를 직접 편집하거나 wiki/에 원본을 넣지 않는 것이 핵심 규칙. 기존의 AI News/, Cinema Studio/, Vibe Coding/ 폴더도 raw source로 취급된다.

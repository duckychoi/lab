Pretext — DOM 없는
텍스트 레이아웃 라이브러리

Midjourney 엔지니어 Cheng Lou가 Claude + Codex로 바이브코딩해서 만든 15KB 오픈소스. 브라우저 최대 병목인 DOM reflow를 완전히 우회한다.

왜 빠른가?

기존 브라우저는 텍스트 크기를 알려면 DOM에 넣고 레이아웃을 강제로 계산해야 한다. Pretext는 수학 계산만으로 같은 결과를 낸다.

DOM 방식 (getBoundingClientRect)
45ms / 500개 텍스트
Pretext layout()
0.09ms / 500개 텍스트 → 500x 빠름

핵심 사실

✓15KB, zero dependency
✓한국어·아랍어·이모지 완벽 지원
✓Canvas / SVG / WebGL 렌더링 지원
✓서버사이드 렌더링 가능 (DOM 불필요)
✓MIT License 오픈소스
✓바이브코딩으로 만든 라이브러리 (상징적)

빠른 시작 가이드

📦
1. 설치
npm 또는 bun으로 설치
npm install @chenglou/pretext

# 또는
bun install @chenglou/pretext
🔧
2. prepare()
텍스트를 한 번 전처리. 결과를 재사용할수록 효율적.
import { prepare } from '@chenglou/pretext'

const prepared = prepare(
'안녕하세요 🚀',
'16px Inter'
)
📐
3. layout()
너비를 주면 높이와 줄수를 즉시 반환. DOM 접근 없음.
import { layout } from '@chenglou/pretext'

const { height, lineCount } =
layout(prepared, 300, 20)

// width=300px, lineHeight=20px
🎮
4. 데모 체험
19개+ 인터랙티브 데모 — 게임, ASCII아트, 물리 텍스트 등
bun install
bun start

# 브라우저에서
# /demos/index 열기
import { prepare, layout } from '@chenglou/pretext'

// 1회 전처리 (폰트 측정 캐시)
const prepared = prepare('안녕하세요! 이 텍스트는 DOM 없이 측정됩니다 🚀', '16px Inter')

// 너비 300px, 줄간격 20px로 레이아웃 계산
const { height, lineCount } = layout(prepared, 300, 20)

console.log(`높이: ${height}px, 줄수: ${lineCount}`)

// 500개 텍스트 배치 처리 → 0.09ms (기존 DOM 방식은 45ms)
import { prepareWithSegments, layoutNextLineRange, materializeLineRange } from '@chenglou/pretext'

const article = '긴 본문 텍스트...'
const prepared = prepareWithSegments(article, '16px Inter')

let cursor = { segmentIndex: 0, graphemeIndex: 0 }

// 이미지 영역을 피해서 텍스트 배치
while (cursor.segmentIndex < prepared.segments.length) {
  const lineWidth = getWidthAtRow(cursor) // 이미지 고려한 너비
  const [range, nextCursor] = layoutNextLineRange(prepared, cursor, lineWidth, 20)
  const line = materializeLineRange(prepared, range)
  renderLine(line) // Canvas / SVG / WebGL 등
  cursor = nextCursor
}
import { richInlineLayout } from '@chenglou/pretext/rich-inline'

// 코드 스팬, 멘션, 칩 등 인라인 요소 포함 레이아웃
const segments = [
  { type: 'text', content: '안녕하세요 ' },
  { type: 'code', content: '@chenglou' },
  { type: 'text', content: '님이 만든 라이브러리입니다' },
]

const result = richInlineLayout(segments, 400, 24)
// 브라우저처럼 공백 처리 + 경계 처리 자동
활용 사례
텍스트가 많은 에디터, 코드 에디터, 채팅 앱, 그래프/차트 라벨, 게임 UI — DOM reflow 없이 실시간으로 텍스트 크기를 계산해야 하는 모든 곳.

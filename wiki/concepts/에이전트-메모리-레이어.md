# 에이전트 메모리 레이어

> [!insight] 핵심 인사이트
> 2026년 4월 기준, 에이전트 인프라는 **"코어 실행 레이어 + 메모리 레이어"** 2계층 구조로 표준화되고 있다. hermes-agent·superpowers 같은 실행 프레임워크가 정착한 이후, 다음 경쟁은 **어떻게 기억하고 맥락을 유지하는가**로 이동 중.

## 주요 구현체

### 그래프 기반 메모리
- [[cognee]] (⭐15,974) — 6줄 코드 API, 엔티티 관계 그래프로 지식 저장. RAG를 넘어선 구조화된 지식 관리
- 특징: `cognee.add() → cognee.cognify() → cognee.search()` 심플 인터페이스

### 세션 연속성 메모리
- [[claude-mem]] (⭐60,473) — AI가 세션 컨텍스트를 압축해 다음 세션에 주입. Claude Code 전용
- 특징: 수동 CLAUDE.md 관리 자동화, 장기 프로젝트 컨텍스트 유지

### 프레임워크 내장 메모리
- [[openai-agents-python]] — OpenAI 공식 SDK, 에이전트 상태 관리 내장
- [[hermes-agent]] — NousResearch 에이전트, 롤플레이 + 메모리 통합

## 메모리 방식 비교

| 방식 | 구현체 | 장점 | 단점 |
|------|--------|------|------|
| 그래프 DB | cognee | 관계 표현, 복잡 쿼리 | 설정 복잡 |
| 벡터 RAG | all-MiniLM | 빠른 검색 | 관계 정보 손실 |
| 파일 기반 | LLM-Wiki | 투명성, 유연성 | 자동화 부족 |
| 세션 압축 | claude-mem | 즉시 적용 | Claude Code 전용 |

## 왜 지금 중요한가

에이전트가 단일 작업을 넘어 **장기 프로젝트, 반복 태스크, 팀 협업**으로 확장되면서 기억 문제가 병목으로 부상. hermes-agent 61K 스타, claude-mem 60K 스타는 모두 "AI가 기억하지 못한다"는 불만에 대한 응답.
